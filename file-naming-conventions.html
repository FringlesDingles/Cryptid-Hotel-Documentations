Core Rules — No Exceptions

Before naming anything

🔡
Lowercase only Godot is case-sensitive on Linux. Always lowercase, no exceptions.
🔗
Underscores only No '-', no spaces. Use '_' as the only separator everywhere.
📁
Type prefix first Always start with the asset type so files group naturally in any folder.
🚫
No vague names No "final", "new", "v2", "test", "copy". Name what it IS.

Sprites & Images

.png · transparent background
Pattern — General
type_name_variant.png — variant is optional
Pattern — Client (one PNG per character, all parts + expressions inside)
client_race_name.png — body parts & expression frames all on one sheet, Godot cuts regions
Pattern — Placeholders (all types)
type_name_placeholder.png — temporary placeholder, will be replaced
type — client, room, item, ui, bg
name — asset name
variant — optional sub-type (icon, layer1…)
placeholder — suffix for all temporary placeholders

item_candle_placeholder.png Item Candle item — placeholder
client_wendigo.png Client Wendigo — all body parts + expression frames on one sheet
Other assets
item_candle.png Item Candle item
bg_forest_layer1.png BG Forest background — parallax layer 1
ui_button_confirm.png UI Confirm button UI element
ui_icon_aura_noise.png UI Noise aura indicator icon
  • ℹ️Client sheets contain everything — body parts for the skeleton AND expression frames for the face. Godot defines regions from a single texture. One file per character, no splits.
  • ⚠️If a second version coexists with the first for a real reason, use _v2 suffix. If v2 replaces v1, simply overwrite — Git keeps the history.
